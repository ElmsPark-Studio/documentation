Install (about 60 seconds)

  1. PageMotor Admin → Plugins → Manage Plugins → Upload a New Plugin.
  2. Upload base.zip first and activate it. This is the shared OpenAI-compatible base.
  3. Upload the provider you want (e.g. deepseek.zip) and activate it.
  4. Go to AI Setup, tick Use Architect AI, pick your provider and model, then open its Settings and paste your API key.
  5. On the Admin dashboard, ask the Architect prompt box to “create a Page about…” and watch it build.

Order matters: a provider only appears under AI Setup once the base plugin is active. Install and activate base.zip first, then the provider.

If you only want the generic route (a local model, or any endpoint not listed), the base plugin alone is enough: it gives you an editable Base URL and Model field.

Cost: on deepseek-v4-flash a full multi-section page runs well under a cent, so a few dollars of credit lasts a long time. Tip: switch on a balance alert in your DeepSeek dashboard, so a dry balance emails you before a build fails with a provider error.

Good to know: review what it writes before you publish, as DeepSeek can get facts and figures wrong. And because hosted DeepSeek processes data on servers in China, keep regulated data, client PII and financials out of it. For those, self-host a model behind the generic base, or use a provider with data-residency guarantees.
